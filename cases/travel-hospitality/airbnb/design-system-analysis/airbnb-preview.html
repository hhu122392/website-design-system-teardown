05 - Known Gaps

Public-only evidence boundaries

This preview uses public Airbnb pages and public asset URLs. It is an independent design-system analysis, not an official Airbnb guideline.

  • The user requested no login, so wishlist, trips, messages, account menus, and host dashboards were not inspected.
  • No search, booking, payment, login, or state-changing action was submitted.
  • Only one public room-detail sample was opened, and it loaded limited detail content in this environment.
  • The Services page exposed limited service-card content in the public state observed.
  • Remote source images and icons are loaded from public Airbnb URLs to satisfy the source-asset requirement.
